status-app

Password protected single user status system for rapidly prototyping mobile apps

Follow @Joegle

Purpose

The purpose of the status-app is to provide the foundations of a status logging system for a mobile device user.

This software repository provides a basic web application that allows a single user to privately and conveniently write a text note and remotely archive it.

The status-app general user story:

A person turns on their mobile phone and visits a domain running an instance of the status-app and types what they are doing at the moment. The user submits the data to be remotely stored.

For technical details see the status-app project page.

Value Know thyself

Maintaining a traditional "paper and pen" journal is a powerful practice and this repo is made to help explore the introspection value that can be further leveraged with the advantages that come with always having a mobile computer .

Journal keeping can benefit us by guiding and refining our focus when we try to encode the present moment into written form [some research: 1, 2]. After a journal entry is created it is also available for possible integration into wisdom. The collection of statuses compose a chronology that can be reviewed to form a feedback loop.

Spaces of interest with this project utilizing a personal feedback loop include:

  • personal data mining
  • personal narative building
  • captology schemes (behavior modification)
  • distributed cognition and expert systems

This study of personal tracking also calls to attention the responsbility that comes with owning your small data (versus outsourced big data) such as computer system security, privacy management and defining what wisdom is and seeking it.

Foundational Software Features

The status-app was designed to meet these basic technical objectives:

  • Allow a user to submit status information to a remote database for storage
  • Mobile focused interface design to decrease note taking "friction"
  • Provide session authentication primitives:
    • Login with password
    • Manual password changing
    • Cookie sessions for a "Remember me"
    • User initiated logout of session
    • Server response messages

For implementation details see the status-app project page.
